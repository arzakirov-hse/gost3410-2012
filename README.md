# ГОСТ Р 34.10-2012 — реализация ЭЦП на Python

Закиров Андрей

Учебная реализация алгоритма электронной цифровой подписи по стандарту **ГОСТ Р 34.10-2012** (256-битный вариант) с использованием хэш-функции **ГОСТ Р 34.11-2012 (Стрибог-256)**.

Вся арифметика эллиптической кривой (сложение точек, удвоение, скалярное умножение, расширенный алгоритм Евклида, обратный элемент по модулю) написана вручную. Внешние криптографические библиотеки не используются — кроме `gostcrypto`, реализующей хэш-функцию Стрибог (это разрешено заданием).

> **Внимание.** Программа предназначена исключительно для учебных целей. Не использовать в реальных системах защиты информации.

---

## Возможности

1. Генерация ключевой пары `(d, Q)`.
2. Ручной ввод закрытого ключа `d` и вычисление соответствующего открытого ключа `Q = d·P`.
3. Формирование подписи:
   - по реальному файлу (хэш считается алгоритмом Стрибог-256);
   - по введённому вручную хэшу (для сверки с эталонными примерами из ГОСТа).
4. Проверка подписи:
   - по реальному файлу;
   - по введённому вручную хэшу;
   - подпись `(r, s)` берётся либо из файла, либо вводится вручную.
5. Возможность задать одноразовое число `k` вручную при подписании — необходимо для воспроизведения детерминированных эталонных примеров.
6. Вывод всех промежуточных значений алгоритма (`x_C`, `y_C`, `r`, `s`, `v`, `z1`, `z2`, `x_C'`, `R`) для контроля корректности и сверки с приложением А стандарта.

## Состав репозитория

| Файл | Описание |
|---|---|
| `gost3410-2012.py` | Основная программа: реализация алгоритма ГОСТ Р 34.10-2012 и интерактивное меню. |
| `priv.txt` | Закрытый ключ `d` в шестнадцатеричном виде (одна строка). Создаётся программой при сохранении сгенерированной ключевой пары. |
| `pub.txt` | Открытый ключ `Q = (x, y)` в шестнадцатеричном виде (две строки: `x` и `y`). Создаётся программой. |
| `sig.txt` | Файл электронной подписи — две строки: `r` и `s` в шестнадцатеричном виде. Создаётся при формировании подписи. |

## Сценарии использования

### Сценарий 1. Подписание реального файла

1. Пункт меню `1` — сгенерировать ключевую пару. Сохранить ключи в `priv.txt` и `pub.txt`.
2. Пункт меню `3` — сформировать подпись:
   - источник хэша: `1` (реальный файл);
   - путь к файлу: `input.txt`;
   - файл закрытого ключа: `priv.txt`;
   - `k`: пустая строка (случайное значение);
   - сохранить подпись в `sig.txt`.
3. Пункт меню `4` — проверить подпись:
   - источник хэша: `1` (реальный файл);
   - путь к файлу: `input.txt`;
   - источник подписи: `1` (файл);
   - файл подписи: `sig.txt`;
   - файл открытого ключа: `pub.txt`.

Результат: **«Подпись ВЕРНА»**. Если изменить хотя бы один байт в `input.txt`, проверка вернёт **«Подпись НЕВЕРНА»**.

### Сценарий 2. Сверка с эталонным примером из приложения А ГОСТ Р 34.10-2012

В стандарте задан искусственный хэш `e`, для которого получить совпадение через хэширование реального файла невозможно (это атака на прообраз хэш-функции). Поэтому в пунктах 3 и 4 предусмотрен ручной ввод хэша.

**Эталонные значения из приложения А ГОСТ Р 34.10-2012:**

| Параметр | Значение |
|---|---|
| `d`  | `0x7A929ADE789BB9BE10ED359DD39A72C11B60961F49397EEE1D19CE9891EC3B28` |
| `Qx` | `0x7F2B49E270DB6D90D8595BEC458B50C58585BA1D4E9B788F6689DBD8E56FD80B` |
| `Qy` | `0x26F1B489D6701DD185C8413A977B3CBBAF64D1C593D26627DFFB101A87FF77DA` |
| `e`  | `0x2DFBC1B372D89A1188C09C52E0EEC61FCE52032AB1022E8E67ECE6672B043EE5` |
| `k`  | `0x77105C9B20BCD3122823C8CF6FCC7B956DE33814E95B7FE64FED924594DCEAB3` |
| `r`  | `0x41AA28D2F1AB148280CD9ED56FEDA41974053554A42767B83AD043FD39DC0493` |
| `s`  | `0x01456C64BA4642A1653C235A98A60249BCD6D3F746B631DF928014F6C5BF9C40` |

**Порядок сверки:**

1. **Пункт меню `2`** — ввести `d` из таблицы. Программа выведет `x` и `y` открытого ключа. Сверить с `Qx` и `Qy`. Сохранить ключи в `priv.txt` и `pub.txt`.
2. **Пункт меню `3`** — сформировать подпись:
   - источник хэша: `2` (ручной ввод), ввести `e`;
   - файл закрытого ключа: `priv.txt`;
   - `k`: ввести значение из таблицы;
   - программа выведет `r` и `s`. Сверить с эталонными.
   - Сохранить подпись в `sig.txt`.
3. **Пункт меню `4`** — проверить подпись:
   - источник хэша: `2` (ручной ввод), ввести `e`;
   - источник подписи: `1` (файл `sig.txt`) или `2` (ввод `r` и `s` вручную);
   - файл открытого ключа: `pub.txt`;
   - результат: **«Подпись ВЕРНА»**.

Если все три шага дали совпадение с эталоном — реализация полностью соответствует стандарту.

## Параметры эллиптической кривой

В программе по умолчанию заданы параметры из контрольного примера приложения А стандарта:

- `p` — модуль поля;
- `a`, `b` — коэффициенты кривой `y² ≡ x³ + a·x + b (mod p)`;
- `q` — порядок циклической подгруппы;
- `(x_P, y_P)` — координаты базовой точки.

Все параметры объявлены в начале файла `gost3410-2012.py` и могут быть заменены на любой другой согласованный набор (например, рекомендованные параметры из ГОСТ Р 34.10-2012 или ГОСТ Р 34.10-2012 RFC 7836).
